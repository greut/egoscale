exo api network create

Creates a network

Synopsis

Creates a network http://cloudstack.apache.org/api/apidocs-4.4/user/createNetwork.html

exo api network create [flags]

Options

--displaynetwork - an optional field, whether to the display the network to the end user or not.
--displaytext - the display text of the network
--domainid - domain ID of the account owning a network
--endip - the ending IP address in the network IP range. Required for managed networks.
--endipv6 - the ending IPv6 address in the IPv6 network range
--gateway - the gateway of the network. Required for Shared networks and Isolated networks when it belongs to VPC
--ip6cidr - the CIDR of IPv6 network, must be at least /64
--ip6gateway - the gateway of the IPv6 network. Required for Shared networks and Isolated networks when it belongs to VPC
--isolatedpvlan - the isolated private vlan for this network
--name - the name of the network
--netmask - the netmask of the network. Required for managed networks.
--networkdomain - network domain
--networkofferingid - (required) the network offering id
--physicalnetworkid - the Physical Network ID the network belongs to
--startip - the beginning IP address in the network IP range. Required for managed networks.
--startipv6 - the beginning IPv6 address in the IPv6 network range
--subdomainaccess - Defines whether to allow subdomains to use networks dedicated to their parent domain(s). Should be used with aclType=Domain, defaulted to allow.subdomain.network.access global config if not specified
--vlan - the ID or VID of the network
--zoneid - (required) the Zone ID for the network
--help, -h - help for create

Options inherited from parent commands

--account, -A - Account to use in config file [env EXOSCALE_ACCOUNT]
--config, -C - Specify an alternate config file [env EXOSCALE_CONFIG]
--debug, -d - debug mode on
--dry-run, -D - produce a cURL ready URL
