exo api network update

Updates a network

Synopsis

Updates a network http://cloudstack.apache.org/api/apidocs-4.4/user/updateNetwork.html

exo api network update [flags]

Options

--changecidr - Force update even if cidr type is different
--customid - an optional field, in case you want to set a custom id to the resource. Allowed to Root Admins only
--displaynetwork - an optional field, whether to the display the network to the end user or not.
--displaytext - the new display text for the network
--endip - the ending IP address in the network IP range. Required for managed networks.
--guestvmcidr - CIDR for Guest VMs,Cloudstack allocates IPs to Guest VMs only from this CIDR
--id - (required) the ID of the network
--name - the new name for the network
--netmask - the netmask of the network. Required for managed networks.
--networkdomain - network domain
--networkofferingid - network offering ID
--startip - the beginning IP address in the network IP range. Required for managed networks.
--help, -h - help for update

Options inherited from parent commands

--account, -A - Account to use in config file [env EXOSCALE_ACCOUNT]
--config, -C - Specify an alternate config file [env EXOSCALE_CONFIG]
--debug, -d - debug mode on
--dry-run, -D - produce a cURL ready URL
