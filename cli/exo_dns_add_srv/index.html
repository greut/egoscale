exo dns add SRV

Add SRV record type to a domain

Synopsis

Add an «SRV» record to specify the location of servers for a specific service.

exo dns add SRV <domain name> [flags]

Options

--help, -h - help for SRV
--name, -n - Leave this blank to create a record for <domain name>, You may use the '*' wildcard here.
--port, -P - The 'TCP' or 'UDP' port on which the service is found.
--priority - Priority
--protocol, -p - This will usually be 'TCP' or 'UDP'.
--symbolic-name, -s - This will be a symbolic name for the service, like 'sip'. It might also be called Service at other DNS providers.
--target - The canonical hostname of the machine providing the service.
--ttl, -t - The time in second to leave (refresh rate) of the record.
--weight, -w - A relative weight for 'SRV' records with the same priority.

Options inherited from parent commands

--account, -A - Account to use in config file [env EXOSCALE_ACCOUNT]
--config, -C - Specify an alternate config file [env EXOSCALE_CONFIG]
