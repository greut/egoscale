cs deployVirtualMachine

Creates and automatically starts a virtual machine based on a service offering, disk offering, and template. http://cloudstack.apache.org/api/apidocs-4.4/user/deployVirtualMachine.html

Flags

--account value – an optional account for the virtual machine. Must be used with domainid.
--affinitygroupids value – comma separated list of affinity groups id that are going to be applied to the virtual machine. Mutually exclusive with affinitygroupnames parameter
--affinitygroupnames value – comma separated list of affinity groups names that are going to be applied to the virtual machine.Mutually exclusive with affinitygroupids parameter
--customid value – an optional field, in case you want to set a custom id to the resource. Allowed to Root Admins only
--deploymentplanner value – Deployment planner to use for vm allocation. Available to ROOT admin only
--details value – used to specify the custom parameters.
--diskofferingid value – the ID of the disk offering for the virtual machine. If the template is of ISO format, the diskofferingid is for the root disk volume. Otherwise this parameter is used to indicate the offering for the data disk volume. If the templateid parameter passed is from a Template object, the diskofferingid refers to a DATA Disk Volume created. If the templateid parameter passed is from an ISO object, the diskofferingid refers to a ROOT Disk Volume created.
--displayname value – an optional user generated name for the virtual machine
--displayvm value – an optional field, whether to the display the vm to the end user or not. (default: unset)
--domainid value – an optional domainid for the virtual machine. If the account parameter is used, domainid must also be used.
--group value – an optional group for the virtual machine
--hostid value – destination Host ID to deploy the VM to - parameter available for root admin only
--hypervisor value – the hypervisor on which to deploy the virtual machine
--ip4 value – True to set an IPv4 to the default interface (default: unset)
--ip6 value – True to set an IPv6 to the default interface (default: unset)
--ip6address value – the ipv6 address for default vm’s network
--ipaddress value – the ip address for default vm’s network
--keyboard value – an optional keyboard device type for the virtual machine. valid value can be one of de,de-ch,es,fi,fr,fr-be,fr-ch,is,it,jp,nl-be,no,pt,uk,us
--keypair value – name of the ssh key pair used to login to the virtual machine
--name value – host name for the virtual machine
--networkids value – list of network ids used by virtual machine. Can’t be specified with ipToNetworkList parameter
--rootdisksize value – Optional field to resize root disk on deploy. Value is in GB. Only applies to template-based deployments. Analogous to details[0].rootdisksize, which takes precedence over this parameter if both are provided (default: 0)
--securitygroupids value – comma separated list of security groups id that going to be applied to the virtual machine. Should be passed only when vm is created from a zone with Basic Network support. Mutually exclusive with securitygroupnames parameter
--securitygroupnames value – comma separated list of security groups names that going to be applied to the virtual machine. Should be passed only when vm is created from a zone with Basic Network support. Mutually exclusive with securitygroupids parameter
--serviceofferingid value – (required) the ID of the service offering for the virtual machine
--size value – the arbitrary size for the DATADISK volume. Mutually exclusive with diskofferingid (default: 0)
--startvm value – true if start vm after creating. Default value is true (default: unset)
--templateid value – (required) the ID of the template for the virtual machine
--userdata value – an optional binary data that can be sent to the virtual machine upon a successful deployment. This binary data must be base64 encoded before adding it to the request. Using HTTP GET (via querystring), you can send up to 2KB of data after base64 encoding. Using HTTP POST(via POST body), you can send up to 32K of data after base64 encoding.
--zoneid value – (required) availability zone for the virtual machine
