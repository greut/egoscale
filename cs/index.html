cs: Command-Line Interface for CloudStack

Like the Pythonic cs but in Go.

Installation

Grab it from the release section.

or build it yourself, it requires somes dependencies though to be pulled using dep.

Configuration

Create a config file cloudstack.ini or $HOME/.cloudstack.ini.

; Default region
[cloudstack]

; Exoscale credential
endpoint = https://api.exoscale.ch/compute
key = EXO...
secret = ...

theme = fruity


; Another region
[cloudstack:production]

endpoint = https://api.exoscale.ch/compute
key = EXO...
secret = ...

theme = vim


; global config for themes
[exoscale]
; Pygments theme, see: https://xyproto.github.io/splash/docs/
; dark
theme = monokai
; light
theme = tango
; no colors (only boldness is allowed)
theme = nocolors

Themes

Thanks to Alec Thomas’ chroma, it supports many themes for your output:

  • https://xyproto.github.io/splash/docs/
  • https://help.farbox.com/pygments.html

Usage

Some of the flags around a command.

$ cs (-h | --help)              global help
$ cs <command> (-h | --help)    help of a specific command
$ cs <command> (-d | --debug)   show the command and its expected output
$ cs <command> (-D | --dry-run) show the signed command
$ cs (-r | --region) <region>   specify a different region, default `cloudstack`

Commands

  • cs activateIp6

    Activate the IPv6 on the VM's nic <http://cloudstack.apache.org/api/apidocs-4.4/user/activateIp6.html>

  • cs addIpToNic

    Assigns secondary IP to NIC <http://cloudstack.apache.org/api/apidocs-4.4/user/addIpToNic.html>

  • cs addNicToVirtualMachine

    Adds VM to specified network by creating a NIC <http://cloudstack.apache.org/api/apidocs-4.4/user/addNicToVirtualMachine.html>

  • cs associateIpAddress

    Acquires and associates a public IP to an account. <http://cloudstack.apache.org/api/apidocs-4.4/user/associateIpAddress.html>

  • cs authorizeSecurityGroupEgress

    Authorizes a particular ingress/egress rule for this security group <http://cloudstack.apache.org/api/apidocs-4.4/user/authorizeSecurityGroupEgress.html>

  • cs authorizeSecurityGroupIngress

    Authorizes a particular ingress/egress rule for this security group <http://cloudstack.apache.org/api/apidocs-4.4/user/authorizeSecurityGroupIngress.html>

  • cs changeServiceForVirtualMachine

    Changes the service offering for a virtual machine. The virtual machine must be in a "Stopped" state for this command to take effect. <http://cloudstack.apache.org/api/apidocs-4.4/user/changeServiceForVirtualMachine.html>

  • cs copyTemplate

    Copies a template from one zone to another. <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/copyTemplate.html>

  • cs createAffinityGroup

    Creates an affinity/anti-affinity group <http://cloudstack.apache.org/api/apidocs-4.4/user/createAffinityGroup.html>

  • cs createInstanceGroup

    Creates a vm group <http://cloudstack.apache.org/api/apidocs-4.4/user/createInstanceGroup.html>

  • cs createNetwork

    Creates a network <http://cloudstack.apache.org/api/apidocs-4.4/user/createNetwork.html>

  • cs createSSHKeyPair

    Create a new keypair and returns the private key <http://cloudstack.apache.org/api/apidocs-4.4/user/createSSHKeyPair.html>

  • cs createSecurityGroup

    Creates a security group <http://cloudstack.apache.org/api/apidocs-4.4/user/createSecurityGroup.html>

  • cs createSnapshot

    Creates an instant snapshot of a volume. <http://cloudstack.apache.org/api/apidocs-4.4/user/createSnapshot.html>

  • cs createTags

    Creates resource tag(s) <http://cloudstack.apache.org/api/apidocs-4.4/user/createTags.html>

  • cs createTemplate

    Creates a template of a virtual machine. The virtual machine must be in a STOPPED state. A template created from this command is automatically designated as a private template visible to the account that created it. <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/createTemplate.html>

  • cs createUser

    Creates a user for an account that already exists <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/createUser.html>

  • cs deleteAffinityGroup

    Deletes affinity group <http://cloudstack.apache.org/api/apidocs-4.4/user/deleteAffinityGroup.html>

  • cs deleteInstanceGroup

    Deletes a vm group <http://cloudstack.apache.org/api/apidocs-4.4/user/deleteInstanceGroup.html>

  • cs deleteNetwork

    Deletes a network <http://cloudstack.apache.org/api/apidocs-4.4/user/deleteNetwork.html>
