cs createTemplate

Creates a template of a virtual machine. The virtual machine must be in a STOPPED state. A template created from this command is automatically designated as a private template visible to the account that created it. http://cloudstack.apache.org/api/apidocs-4.4/root_admin/createTemplate.html

Flags

--bits value – 32 or 64 bit (default: 0)
--details value – Template details in key/value pairs.
--displaytext value – (required) the display text of the template. This is usually used for display purposes.
--isdynamicallyscalable value – true if template contains XS/VMWare tools inorder to support dynamic scaling of VM cpu/memory (default: unset)
--isfeatured value – true if this template is a featured template, false otherwise (default: unset)
--ispublic value – true if this template is a public template, false otherwise (default: unset)
--name value – (required) the name of the template
--ostypeid value – (required) the ID of the OS Type that best represents the OS of this template.
--passwordenabled value – true if the template supports the password reset feature; default is false (default: unset)
--requireshvm value – true if the template requres HVM, false otherwise (default: unset)
--snapshotid value – the ID of the snapshot the template is being created from. Either this parameter, or volumeId has to be passed in
--templatetag value – the tag for this template.
--url value – Optional, only for baremetal hypervisor. The directory name where template stored on CIFS server
--virtualmachineid value – Optional, VM ID. If this presents, it is going to create a baremetal template for VM this ID refers to. This is only for VM whose hypervisor type is BareMetal
--volumeid value – the ID of the disk volume the template is being created from. Either this parameter, or snapshotId has to be passed in

Global Flags

--debug, -d – debug mode on
--region value, -r value – cloudstack.ini file section name
--theme value, -t value – syntax highlighting theme, see: https://xyproto.github.io/splash/docs/
--help, -h – show help
--version, -v – print the version
