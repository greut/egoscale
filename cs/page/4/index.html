cs: Command-Line Interface for CloudStack

Like the Pythonic cs but in Go.

Installation

Grab it from the release section.

or build it yourself, it requires somes dependencies though to be pulled using dep.

Configuration

Create a config file cloudstack.ini or $HOME/.cloudstack.ini.

; Default region
[cloudstack]

; Exoscale credential
endpoint = https://api.exoscale.ch/compute
key = EXO...
secret = ...

theme = fruity


; Another region
[cloudstack:production]

endpoint = https://api.exoscale.ch/compute
key = EXO...
secret = ...

theme = vim


; global config for themes
[exoscale]
; Pygments theme, see: https://xyproto.github.io/splash/docs/
; dark
theme = monokai
; light
theme = tango
; no colors (only boldness is allowed)
theme = nocolors

Themes

Thanks to Alec Thomas’ chroma, it supports many themes for your output:

  • https://xyproto.github.io/splash/docs/
  • https://help.farbox.com/pygments.html

Usage

Some of the flags around a command.

$ cs (-h | --help)              global help
$ cs <command> (-h | --help)    help of a specific command
$ cs <command> (-d | --debug)   show the command and its expected output
$ cs <command> (-D | --dry-run) show the signed command
$ cs (-r | --region) <region>   specify a different region, default `cloudstack`

Commands

  • cs prepareTemplate

    load template into primary storage <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/prepareTemplate.html>

  • cs queryAsyncJobResult

    Retrieves the current status of asynchronous job. <http://cloudstack.apache.org/api/apidocs-4.4/user/queryAsyncJobResult.html>

  • cs queryReverseDnsForPublicIpAddress

    Query the PTR DNS record for the public IP address <http://cloudstack.apache.org/api/apidocs-4.4/user/queryReverseDnsForPublicIpAddress.html>

  • cs queryReverseDnsForVirtualMachine

    Query the PTR DNS record(s) for the virtual machine <http://cloudstack.apache.org/api/apidocs-4.4/user/queryReverseDnsForVirtualMachine.html>

  • cs rebootVirtualMachine

    Reboots a virtual machine. <http://cloudstack.apache.org/api/apidocs-4.4/user/rebootVirtualMachine.html>

  • cs recoverVirtualMachine

    Recovers a virtual machine. <http://cloudstack.apache.org/api/apidocs-4.4/user/recoverVirtualMachine.html>

  • cs registerSSHKeyPair

    Register a public key in a keypair under a certain name <http://cloudstack.apache.org/api/apidocs-4.4/user/registerSSHKeyPair.html>

  • cs registerTemplate

    Registers an existing template into the CloudStack cloud. <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/registerTemplate.html>

  • cs registerUserKeys

    This command allows a user to register for the developer API, returning a secret key and an API key. This request is made through the integration API port, so it is a privileged command and must be made on behalf of a user. It is up to the implementer just how the username and password are entered, and then how that translates to an integration API request. Both secret key and API key should be returned to the user <http://cloudstack.apache.org/api/apidocs-4.4/user/registerUserKeys.html>

  • cs removeIpFromNic

    Removes secondary IP from the NIC. <http://cloudstack.apache.org/api/apidocs-4.4/user/removeIpFromNic.html>

  • cs removeNicFromVirtualMachine

    Removes VM from specified network by deleting a NIC <http://cloudstack.apache.org/api/apidocs-4.4/user/removeNicFromVirtualMachine.html>

  • cs resetPasswordForVirtualMachine

    Resets the password for virtual machine. The virtual machine must be in a "Stopped" state and the template must already support this feature for this command to take effect. <http://cloudstack.apache.org/api/apidocs-4.4/user/resetPasswordForVirtualMachine.html>

  • cs resetSSHKeyForVirtualMachine

    Resets the SSH Key for virtual machine. The virtual machine must be in a "Stopped" state. <http://cloudstack.apache.org/api/apidocs-4.4/user/resetSSHKeyForVirtualMachine.html>

  • cs resizeVolume

    Resizes a volume <http://cloudstack.apache.org/api/apidocs-4.4/user/resizeVolume.html>

  • cs restartNetwork

    Restarts the network; includes 1) restarting network elements - virtual routers, dhcp servers 2) reapplying all public ips 3) reapplying loadBalancing/portForwarding rules <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/restartNetwork.html>

  • cs restoreVirtualMachine

    Restore a VM to original template/ISO or new template/ISO <http://cloudstack.apache.org/api/apidocs-4.4/user/restoreVirtualMachine.html>

  • cs revertSnapshot

    revert a volume snapshot. <http://cloudstack.apache.org/api/apidocs-4.4/user/revertSnapshot.html>

  • cs revokeSecurityGroupEgress

    Deletes a particular egress rule from this security group <http://cloudstack.apache.org/api/apidocs-4.4/user/revokeSecurityGroupEgress.html>

  • cs revokeSecurityGroupIngress

    Deletes a particular ingress rule from this security group <http://cloudstack.apache.org/api/apidocs-4.4/user/revokeSecurityGroupIngress.html>

  • cs scaleVirtualMachine

    Scales the virtual machine to a new service offering. <http://cloudstack.apache.org/api/apidocs-4.4/user/scaleVirtualMachine.html>
