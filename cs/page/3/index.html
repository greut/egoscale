cs: Command-Line Interface for CloudStack

Like the Pythonic cs but in Go.

Installation

Grab it from the release section.

or build it yourself, it requires somes dependencies though to be pulled using dep.

Configuration

Create a config file cloudstack.ini or $HOME/.cloudstack.ini.

; Default region
[cloudstack]

; Exoscale credential
endpoint = https://api.exoscale.ch/compute
key = EXO...
secret = ...

theme = fruity


; Another region
[cloudstack:production]

endpoint = https://api.exoscale.ch/compute
key = EXO...
secret = ...

theme = vim


; global config for themes
[exoscale]
; Pygments theme, see: https://xyproto.github.io/splash/docs/
; dark
theme = monokai
; light
theme = tango
; no colors (only boldness is allowed)
theme = nocolors

Themes

Thanks to Alec Thomas’ chroma, it supports many themes for your output:

  • https://xyproto.github.io/splash/docs/
  • https://help.farbox.com/pygments.html

Usage

Some of the flags around a command.

$ cs (-h | --help)              global help
$ cs <command> (-h | --help)    help of a specific command
$ cs <command> (-d | --debug)   show the command and its expected output
$ cs <command> (-D | --dry-run) show the signed command
$ cs (-r | --region) <region>   specify a different region, default `cloudstack`

Commands

  • cs listHosts

    Lists hosts. <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/listHosts.html>

  • cs listInstanceGroups

    Lists vm groups <http://cloudstack.apache.org/api/apidocs-4.4/user/listInstanceGroups.html>

  • cs listNetworkOfferings

    Lists all available network offerings. <http://cloudstack.apache.org/api/apidocs-4.4/user/listNetworkOfferings.html>

  • cs listNetworks

    Lists all available networks. <http://cloudstack.apache.org/api/apidocs-4.4/user/listNetworks.html>

  • cs listNics

    list the vm nics IP to NIC <http://cloudstack.apache.org/api/apidocs-4.4/user/listNics.html>

  • cs listOsCategories

    Lists all supported OS categories for this cloud. <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/listOsCategories.html>

  • cs listPublicIpAddresses

    Lists all public ip addresses <http://cloudstack.apache.org/api/apidocs-4.4/user/listPublicIpAddresses.html>

  • cs listResourceDetails

    List resource detail(s) <http://cloudstack.apache.org/api/apidocs-4.4/user/listResourceDetails.html>

  • cs listResourceLimits

    Lists resource limits. <http://cloudstack.apache.org/api/apidocs-4.4/user/listResourceLimits.html>

  • cs listSSHKeyPairs

    List registered keypairs <http://cloudstack.apache.org/api/apidocs-4.4/user/listSSHKeyPairs.html>

  • cs listSecurityGroups

    Lists security groups <http://cloudstack.apache.org/api/apidocs-4.4/user/listSecurityGroups.html>

  • cs listServiceOfferings

    Lists all available service offerings. <http://cloudstack.apache.org/api/apidocs-4.4/user/listServiceOfferings.html>

  • cs listSnapshots

    Lists all available snapshots for the account. <http://cloudstack.apache.org/api/apidocs-4.4/user/listSnapshots.html>

  • cs listTags

    List resource tag(s) <http://cloudstack.apache.org/api/apidocs-4.4/user/listTags.html>

  • cs listTemplates

    List all public, private, and privileged templates. <http://cloudstack.apache.org/api/apidocs-4.4/user/listTemplates.html>

  • cs listUsers

    Lists user accounts <http://cloudstack.apache.org/api/apidocs-4.4/user/listUsers.html>

  • cs listVirtualMachines

    List the virtual machines owned by the account. <http://cloudstack.apache.org/api/apidocs-4.4/user/listVirtualMachines.html>

  • cs listVolumes

    Lists all volumes. <http://cloudstack.apache.org/api/apidocs-4.4/user/listVolumes.html>

  • cs listZones

    Lists zones <http://cloudstack.apache.org/api/apidocs-4.4/user/listZones.html>

  • cs migrateVirtualMachine

    Attempts Migration of a VM to a different host or Root volume of the vm to a different storage pool <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/migrateVirtualMachine.html>
