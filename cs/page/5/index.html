cs: Command-Line Interface for CloudStack

Like the Pythonic cs but in Go.

Installation

Grab it from the release section.

or build it yourself, it requires somes dependencies though to be pulled using dep.

Configuration

Create a config file cloudstack.ini or $HOME/.cloudstack.ini.

; Default region
[cloudstack]

; Exoscale credential
endpoint = https://api.exoscale.ch/compute
key = EXO...
secret = ...

theme = fruity


; Another region
[cloudstack:production]

endpoint = https://api.exoscale.ch/compute
key = EXO...
secret = ...

theme = vim


; global config for themes
[exoscale]
; Pygments theme, see: https://xyproto.github.io/splash/docs/
; dark
theme = monokai
; light
theme = tango
; no colors (only boldness is allowed)
theme = nocolors

Themes

Thanks to Alec Thomas’ chroma, it supports many themes for your output:

  • https://xyproto.github.io/splash/docs/
  • https://help.farbox.com/pygments.html

Usage

Some of the flags around a command.

$ cs (-h | --help)              global help
$ cs <command> (-h | --help)    help of a specific command
$ cs <command> (-d | --debug)   show the command and its expected output
$ cs <command> (-D | --dry-run) show the signed command
$ cs (-r | --region) <region>   specify a different region, default `cloudstack`

Commands

  • cs startVirtualMachine

    Starts a virtual machine. <http://cloudstack.apache.org/api/apidocs-4.4/user/startVirtualMachine.html>

  • cs stopVirtualMachine

    Stops a virtual machine. <http://cloudstack.apache.org/api/apidocs-4.4/user/stopVirtualMachine.html>

  • cs updateDefaultNicForVirtualMachine

    Changes the default NIC on a VM <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/updateDefaultNicForVirtualMachine.html>

  • cs updateHost

    Updates a host. <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/updateHost.html>

  • cs updateInstanceGroup

    Updates a vm group <http://cloudstack.apache.org/api/apidocs-4.4/user/updateInstanceGroup.html>

  • cs updateIpAddress

    Updates an ip address <http://cloudstack.apache.org/api/apidocs-4.4/user/updateIpAddress.html>

  • cs updateNetwork

    Updates a network <http://cloudstack.apache.org/api/apidocs-4.4/user/updateNetwork.html>

  • cs updateReverseDnsForPublicIpAddress

    Update/create the PTR DNS record for the public IP address <http://cloudstack.apache.org/api/apidocs-4.4/user/updateReverseDnsForPublicIpAddress.html>

  • cs updateReverseDnsForVirtualMachine

    Update/create the PTR DNS record(s) for the virtual machine <http://cloudstack.apache.org/api/apidocs-4.4/user/updateReverseDnsForVirtualMachine.html>

  • cs updateUser

    Updates a user account <http://cloudstack.apache.org/api/apidocs-4.4/root_admin/updateUser.html>

  • cs updateVMAffinityGroup

    Updates the affinity/anti-affinity group associations of a virtual machine. The VM has to be stopped and restarted for the new properties to take effect. <http://cloudstack.apache.org/api/apidocs-4.4/user/updateVMAffinityGroup.html>

  • cs updateVirtualMachine

    Updates properties of a virtual machine. The VM has to be stopped and restarted for the new properties to take effect. UpdateVirtualMachine does not first check whether the VM is stopped. Therefore, stop the VM manually before issuing this call. <http://cloudstack.apache.org/api/apidocs-4.4/user/updateVirtualMachine.html>

  • cs updateVmNicIp

    Update the default Ip of a VM Nic <http://cloudstack.apache.org/api/apidocs-4.4/user/updateVmNicIp.html>
